# 🎨 宝可梦 ASCII 美术规范 + 帧契约(画师必读铁律)

> 这份规范是给"画师"(为某只宝可梦产出 15 状态 × 4 帧 ASCII 动画)的**强制契约**。
> 目标:每只宝可梦都要画得**和皮卡丘黄金样板同等精致**——毛茸茸、有强物种辨识度、
> 切帧不抖、配色协议正确。不达标会被打回重画。

---

## 0. 黄金样板:先吃透皮卡丘

动手前**必须先读** `pokedex/pikachu.py` 里的 15 组帧。它是审美基准。注意它的精髓:

- **强辨识度的物种轮廓**:毛茸茸波浪耳 `(\~~/)`、红脸颊 `·`、闪电尾 `_/⚡`、小爪 `(")(")`。
  一眼就知道是皮卡丘。**你画的宝可梦也必须一眼可辨**——把该物种最标志性的特征
  (妙蛙的背上球茎/花、小火龙的尾焰、杰尼龟的龟壳、忍者蛙的舌头围巾、路卡利欧的波导犬耳+胸刺)
  做成轮廓的核心。
- **脸是灵魂**:用括号+眼睛组成表情 `(o.o·)` `(^o^·)` `(>_<·)` `(-.-·)` `(T_T·)`,
  每个状态表情不同(开心眯眼、惊讶瞪眼、睡觉闭眼、沮丧流泪)。
- **4 帧成循环动画**:同一状态的 4 帧之间有**细微差异**(眨眼、尾巴摆、嘴开合、身体起伏),
  连起来像在动。不是 4 张一样的图。

## 1. 硬性结构契约(违反 = 直接打回)

### 1.1 必须正好 15 个状态,每个正好 4 帧

状态键(一字不差,全小写):
```
idle  walk_right  walk_left  eat  sleep  happy
jump  look  sing  yawn  surprise  struggle  think  cheer  sad
```

| 状态 | 含义 | 表情/动作要点 |
|---|---|---|
| `idle` | 待机 | 正面,眨眼,尾巴轻摆;最常出现,要最耐看 |
| `walk_right` | 向右跑 | **脸朝右**,四肢交替,身体起伏 |
| `walk_left` | 向左跑 | **脸朝左**(walk_right 的镜像方向) |
| `eat` | 吃东西 | 捧着果子啃,嘴动,旁边写 `nom`/`yum~` |
| `sleep` | 睡觉 | 闭眼 `(-.-)`/`(u.u)`,头顶飘 `Z z` |
| `happy` | 开心 | 张嘴笑 `(^o^)`,举手,迸发元素符号 |
| `jump` | 跳跃 | 蹲→起跳→腾空→落地 4 帧,有上升感 |
| `look` | 东张西望 | 眼睛/脸左右瞟,头顶 `?`/`→`/`←` |
| `sing` | 哼歌 | 头顶音符 `♪`/`♫` 飘动,身体晃 |
| `yawn` | 打哈欠 | 嘴越张越大 `(o○o)`→`(=○=)`,过渡到困 |
| `surprise` | 被戳惊吓 | 缩起来 `(>_<)`→瞪眼 `(O_O)`,抖一抖 |
| `struggle` | 被拖拽挣扎 | 手脚乱蹬 `\(...)/`,喊 `~wah~` |
| `think` | 思考 | 挠头,头顶 `?`/`? ?`,写 `hmm..` |
| `cheer` | 大庆祝 | 比 happy 更夸张,全身迸发元素符号,写 `DONE!` |
| `sad` | 沮丧 | 耷拉,头顶乌云 `☁☁☁`,流泪 `(T_T)`/`💧` |

### 1.2 字符渲染协议(决定上色,务必遵守)

渲染器逐字符上色,规则**写死**如下:

- `·`(U+00B7 中点)→ 渲染成**脸颊色**。每只都要在脸上保留 `·` 当脸颊红点
  (放在眼睛旁,如 `(o.o·)`)。这是"萌点",所有宝可梦统一保留。
- **元素符号**(每只一个,见 §2)→ 渲染成**元素色**。在 happy/cheer/walk 等帧里点缀。
- **其余所有可见字符** → 渲染成**主体色**。

> 所以:皮卡丘把 `⚡` 当元素符号;火系把 `🔥`、水系把 `💧`、草系把 `🌿`、
> 波导把 `✨` 当元素符号。画帧时把该符号放在合适位置(尾巴/迸发/头顶)。

### 1.3 脚部约定

- **静态/趴着的状态**(`idle` `eat` `look` `yawn` `think` `sad` `sleep`):
  下半身用一行**花草 emoji**代替脚:`🍀 🌷 🍀 🌼 🍀`(emoji 间留空格,务必照抄这一行)。
- **会动/站立的状态**(`walk_right` `walk_left` `jump` `happy` `sing` `surprise`
  `struggle` `cheer`):露出爪子/脚,用 `(")(")` 或 `(")  (")` 之类。
  > 例外:四足宝可梦(妙蛙种子/草、利欧路趴着时)可用更贴合的四足脚型,但要统一。

### 1.4 emoji 宽度铁律(避免错位)

渲染器**按单宽列**逐字符定位(`col_i * 字符宽`)。emoji 是**双宽**,会占两列视觉宽但
代码里算一列 → **含 emoji 的行会和上下行错位**。所以:

- **含 emoji 的行,整行只放 emoji + 空格**(像花草行 `🍀 🌷 🍀 🌼 🍀`),
  **不要**在同一行里把 emoji 和单宽字符(字母/括号)混排,否则把上面的脸顶歪。
- 元素符号若是 emoji(🔥💧🌿✨等),尽量**单独成位**或放在行尾,别夹在脸中间。
  皮卡丘的 `⚡` 是个特例(它在多数字体里近似单宽),其它 emoji 没这待遇——
  **稳妥起见,元素符号也尽量靠行首/行尾独立摆放**。
- ASCII 标点/字母(`Z z ♪ ♫ ? ! ~ nom`)是单宽,可自由混排。

### 1.5 对齐与规整

- 每个状态的 4 帧**不需要你手动补齐**等宽——数据包里有 `_normalize()` 会自动把
  4 帧补成等行等列。但**单帧内部**的字符位置要你画准(脸、身、脚上下对齐)。
- 帧用 Python **raw 三引号字符串** `r"""..."""`,首尾各留一个换行(照抄皮卡丘写法)。
- 整体尺寸参考皮卡丘:**3~6 行高、10~16 列宽**。别画太大(桌面占地)或太小(没细节)。

## 2. 各属性的元素符号 + 配色(画师按分配的宝可梦取用)

| 属性 | 元素符号 | 主体色参考 | 脸颊色参考 |
|---|---|---|---|
| 电(皮卡丘) | `⚡` | (255,213,30) 雷电黄 | (232,60,60) 红 |
| 草/毒(妙蛙系) | `🌿` | 妙蛙种子青绿、妙蛙花更深绿 | (220,90,90) |
| 火(小火龙系) | `🔥` | 橙(244,160,90)→喷火龙更深橙 | (230,90,70) |
| 水(杰尼龟系) | `💧` | 浅蓝(110,180,225)→水箭龟深蓝 | (235,150,90) |
| 水/恶(呱呱泡蛙系) | `💧`(忍者蛙可用 `🌀`) | 浅蓝→忍者蛙深藏蓝 | 按各只 |

> 精确 RGB / 图鉴编号 / 每只的独特视觉标志,会在派发时随该只的"特征卡"一起给你。

## 3. 产出格式(画师必须按此返回)

返回一段 **Python 代码**,定义 15 个帧常量 + 一个 `STATES` 引用,格式**严格照抄
皮卡丘样板的写法**(`IDLE = [ r"""...""", ... ]` 共 15 组,每组 4 帧)。例如:

```python
IDLE = [
    r"""
 <第1帧多行字符画>
""",
    r"""
 <第2帧>
""",
    r"""
 <第3帧>
""",
    r"""
 <第4帧>
""",
]
# ... 其余 14 个状态同理 ...
```

**不要**写 `PACK` dict、不要写人设台词配色(那些我来组装)——你只产出**15 组帧常量**。
状态常量名用大写:`IDLE WALK_RIGHT WALK_LEFT EAT SLEEP HAPPY JUMP LOOK SING YAWN
SURPRISE STRUGGLE THINK CHEER SAD`。
